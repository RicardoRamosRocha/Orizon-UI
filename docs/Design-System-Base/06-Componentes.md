# Componentes

## Objetivo

Definir os componentes fundamentais que formarão a primeira versão do Orizon UI.

Todos os componentes deverão utilizar exclusivamente os Design Tokens e seguir o padrão visual estabelecido pelo Design System.

---

# Componentes da Sprint

## Button

Suporte para:

* Primary
* Secondary
* Success
* Warning
* Danger
* Ghost
* Outline
* Link

Estados:

* Normal
* Hover
* Active
* Focus
* Disabled
* Loading

---

## Card

Utilizado para organização visual das informações.

Recursos:

* Header
* Body
* Footer
* Ações
* Variações de elevação

---

## Badge

Utilizado para representar estados e categorias.

Tipos:

* Primary
* Secondary
* Success
* Warning
* Danger
* Info

---

## Alert

Mensagens informativas para o usuário.

Tipos:

* Success
* Warning
* Danger
* Info

---

## Avatar

Representação visual do usuário.

Recursos:

* Imagem
* Iniciais
* Ícone padrão
* Diferentes tamanhos

---

## Divider

Separador visual entre conteúdos.

Suporte para orientação:

* Horizontal
* Vertical

---

## Spinner

Indicador de carregamento.

Variações:

* Pequeno
* Médio
* Grande

---

## Progress

Indicador de progresso.

Tipos:

* Linear
* Circular (estrutura preparada)

---

# Requisitos Gerais

Todos os componentes deverão possuir:

* Responsividade
* Estados visuais consistentes
* Suporte à acessibilidade
* Compatibilidade com tema claro e futura expansão para tema escuro
* API simples e reutilizável em projetos ASP.NET Core MVC Razor

---

# Evolução

Os componentes desta sprint servirão como base para elementos mais avançados nas próximas sprints, como formulários, tabelas, modais, navegação, gráficos e componentes específicos de negócio.
