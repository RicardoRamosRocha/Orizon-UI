# Responsividade

## Objetivo

Garantir que todos os componentes e layouts do Orizon UI sejam desenvolvidos seguindo a abordagem **Mobile First**, oferecendo uma experiência consistente em diferentes tamanhos de tela.

---

# Breakpoints

| Dispositivo  | Faixa           |
| ------------ | --------------- |
| Mobile       | até 575px       |
| Small Tablet | 576px – 767px   |
| Tablet       | 768px – 991px   |
| Notebook     | 992px – 1199px  |
| Desktop      | 1200px – 1399px |
| Wide Screen  | 1400px+         |

---

# Diretrizes

Todos os componentes deverão:

* Adaptar largura automaticamente.
* Evitar rolagem horizontal.
* Utilizar Grid e Flexbox.
* Possuir espaçamentos responsivos.
* Suportar interação por toque.

---

# Layout

## Desktop

* Sidebar fixa
* Conteúdo expandido
* Menus completos

## Tablet

* Sidebar recolhida
* Conteúdo otimizado

## Mobile

* Sidebar Overlay
* Menus simplificados
* Componentes em largura total

---

# Componentes

Os componentes deverão adaptar automaticamente:

* Padding
* Margens
* Fontes
* Ícones
* Altura dos controles
* Quantidade de colunas

---

# Objetivo Final

Toda aplicação construída com o Orizon UI deverá funcionar de forma consistente em smartphones, tablets, notebooks e monitores de alta resolução sem necessidade de CSS específico por projeto.
