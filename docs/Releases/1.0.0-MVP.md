# Orizon UI 1.0.0 MVP Release Notes

## Objective

Prepare Orizon UI as a reusable Razor Class Library for ASP.NET Core MVC/Razor applications, with a local NuGet package and validated static web assets.

## Features

- Design System tokens, typography, utilities, dark mode, and responsive foundations.
- Admin Layout with sidebar, topbar, breadcrumb, and footer.
- Components: Buttons, Cards, Alerts, Badges, Forms, Tables, Modal, Dropdown, Tabs, Pagination, Avatar, Spinner, Progress, Empty State, and Toast.
- Sandbox catalog at `/components`.
- Accessibility improvements for labels, ARIA attributes, keyboard navigation, focus states, modal Escape behavior, dropdown navigation, and tabs.

## Requirements

- .NET 8 SDK.
- ASP.NET Core MVC or Razor Pages consumer application.

## Installation

Generate the local package:

```bash
dotnet pack src/Orizon.UI/Orizon.UI.csproj --configuration Release --output artifacts/packages
```

Install from the local package source:

```bash
dotnet add package Orizon.UI --version 1.0.0-mvp --source artifacts/packages
```

## Usage

Register Tag Helpers:

```razor
@addTagHelper *, Orizon.UI
```

Reference static web assets:

```html
<link rel="stylesheet" href="~/_content/Orizon.UI/css/orizon.css" />
<script type="module" src="~/_content/Orizon.UI/js/orizon.js"></script>
```

Minimal component:

```razor
<orizon-button variant="primary">Save</orizon-button>
```

## Known Limitations

- The repository currently has no root `LICENSE` file; package metadata uses the `MIT` license expression and should be confirmed before public publication.
- The package is validated locally only and is not published to NuGet.org by this sprint.
- Icon rendering depends on the consuming application loading a compatible icon font when icon class names such as `ph ph-plus` are used.

## Validation Checklist

- `dotnet format Orizon.UI.sln`
- `dotnet restore`
- `dotnet build Orizon.UI.sln --configuration Release`
- `dotnet test Orizon.UI.sln --configuration Release`
- `dotnet pack src/Orizon.UI/Orizon.UI.csproj --configuration Release --output artifacts/packages`
- Local consumer application restore/build/run.
- Static web assets under `_content/Orizon.UI`.
- Sandbox routes returning HTTP 200.
