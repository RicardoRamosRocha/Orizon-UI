# Checklist de adoção

- [ ] Importar `orizon.css` e `orizon.js`.
- [ ] Informar rótulos e títulos específicos do contexto.
- [ ] Usar identificadores únicos em drawers, comandos, cartões e colunas.
- [ ] Validar todos os fluxos somente com teclado.
- [ ] Testar tema claro, escuro e contraste.
- [ ] Testar desktop, tablet e mobile sem overflow.
- [ ] Verificar retorno de foco em overlays.
- [ ] Escutar `orizon:kanban-change` quando o domínio precisar persistir no servidor.
- [ ] Tratar indisponibilidade de armazenamento local como preferência não persistida.
- [ ] Validar conteúdo dinâmico com leitor de tela.
